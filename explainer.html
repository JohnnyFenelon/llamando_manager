❓ Problem Statement & Target Audience

Target Audience: Outbound Call Centers, Inside Sales Teams, and Customer Acquisition hubs managing warm or cold leads against fixed payroll hours and ambitious monthly sales targets.

The Core Challenge: Sales supervisors struggle to align lead purchases, daily traffic patterns, and global agent availability (across Morning, Afternoon, Night, and Weekend shifts) into a coherent schedule that guarantees they hit their monthly conversions. Simultaneously, agents are slowed down by complex, cluttered CRM databases and lack quick tools to handle objections on the phone.

How This App Solves It:

  • Bedrock Shift Optimizer: Supervisors input agent constraints and lead targets. Bedrock automatically schedules shifts and outputs a daily outbound call frequency forecast to hit the quota.
  • Minimalist CRM & Dialing: Focuses on key metrics (Calls, Duration, Wins) and features a clean table where agents click "Call" to route phone numbers instantly to the **Amazon Connect Contact Control Panel (CCP)**.
  • AI Sales objections Coach: Directly within the dashboard, agents can prompt Claude-3.5 to draft pitches or handle tough customer objections (like "no budget").

🔒 Code Snippet: Bedrock Scheduler & RLS Policies

Below is the structured Node.js call executed in our server routes to query Amazon Bedrock for the agent schedules and forecast monthly plans:

import { BedrockRuntimeClient, InvokeModelCommand } from "@aws-sdk/client-bedrock-runtime";

const bedrock = new BedrockRuntimeClient({ region: "us-east-1" });

export async function POST(req) {
  const body = await req.json();
  
  // Format prompt based on user's shift parameters and pursuit targets
  const prompt = `System: You are an expert call center shift optimizer...
    Weekly Traffic: \${body.weeklyUsersTarget}
    Agents: [Morning: \${body.agentsMorning}, Afternoon: \${body.agentsAfternoon}, Night: \${body.agentsNight}]
    Target quota: \${body.pursueTarget} sales
    Leads purchased: \${body.leadsPurchased}`;

  const command = new InvokeModelCommand({
    modelId: "anthropic.claude-3-5-sonnet-20241022-v2:0",
    body: JSON.stringify({
      anthropic_version: "bedrock-2023-05-31",
      max_tokens: 1200,
      messages: [{ role: "user", content: prompt }]
    })
  });

  const response = await bedrock.send(command);
  return Response.json(JSON.parse(new TextDecoder().decode(response.body)));
}
